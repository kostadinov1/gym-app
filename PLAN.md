**Goal:** SQLite database with system exercises seeded, ready before any ghost user flow.

**Packages:**
```bash
npm install drizzle-orm expo-sqlite
npm install -D drizzle-kit
```

**Files to create:**
- `src/db/schema.ts` — 6 tables (exercises, workout_plans, workout_routines, routine_exercises, workout_sessions, session_sets)
- `src/db/client.ts` — open SQLite, run drizzle migrations, export `db` handle

- `src/db/migrations/` — generated by `npx drizzle-kit generate`
- `assets/system_exercises.json` — copy from `gym-backend/app/data/system_exercises.json`

**Key schema notes:**
- All IDs: `text` (UUID strings, generated client-side with `crypto.randomUUID()`)

- `exercises.user_id`: `null` = system template, `'local'` = ghost user's custom exercise

**Verify:** Open app → SQLite initializes → 200+ system exercises seeded ✓

## Phase 2 — Auth Flow Changes

**Goal:** "Continue as Guest" button works; guest state persists across app restarts.

**Files to modify:**
- `src/context/AuthContext.tsx` — add `isGuest`, `guestSignIn()`, `promoteGuest(token)`
- `App.tsx` line 54 — `return (userToken || isGuest) ? <RootNavigator /> : <LoginScreen />`
- `src/screens/auth/LoginScreen.tsx` — add "Continue as Guest" button

**AuthContext changes:**
```typescript
// New interface additions
isGuest: boolean;
guestSignIn: () => Promise<void>;       // AsyncStorage.setItem('isGuestMode', 'true')
promoteGuest: (token: string) => Promise<void>;  // queryClient.clear() FIRST, then store JWT

// Bootstrap change
const token = await SecureStore.getItemAsync('userToken');
if (token) setUserToken(token);
else {
  const guest = await AsyncStorage.getItem('isGuestMode');
  if (guest === 'true') setIsGuest(true);
}

// signOut — also clear isGuestMode + prompt to wipe SQLite
```

**CRITICAL in `promoteGuest()`:** Call `queryClient.clear()` BEFORE setting state — flushes local cache to prevent ghost data showing after promotion.

## Phase 9 — Ghost → Registered Migration

**Goal:** Ghost user registers, all local data migrates to cloud, app seamlessly switches to remote mode.

**Backend files to create:**
- `gym-backend/app/schemas/migration.py` — `GuestMigrationPayload`, `MigrationResult` Pydantic models
- `gym-backend/app/routers/migration.py` — `POST /migrate` endpoint

**Register in `gym-backend/app/main.py`:**
```python
from app.routers import migration
app.include_router(migration.router)
```

**Migration algorithm (single DB transaction):**
1. Custom exercises → create server-side → build `local_id → server_id` map
2. System exercises → look up by `slug` → map IDs (don't recreate)
3. Plans + routines + routine_exercises → create with remapped IDs (name conflict → append "(Imported)")
4. Sessions + sets → create with remapped IDs
5. Return `{ id_map, counts, errors[] }`

**Frontend files to create:**
- `src/services/GhostMigrationService.ts` — collects SQLite data, POSTs to `/migrate`

**UI flow:** ProfileScreen "Create Account" → register form → "Syncing your data…" modal → success → `promoteGuest(token)` → user enters Pro trial

**Idempotency:** Store `{ migrated_at, server_user_id }` in SQLite after success. Check before re-migrating.

**Verify:** Ghost data migrates fully ✓; migration runs twice → no duplicates ✓; network fails mid-migration → retry works ✓

## Architecture Reference

### Storage Flow
```
App launch
  ├─ isGuest = true  →  StorageContext provides LocalService (SQLite)
  └─ userToken set   →  StorageContext provides RemoteService (API)

All screens: useStorage().getPlans() / finishWorkout() / etc.
```

### Entitlement Flow
```
Ghost user:    RevenueCat not initialized → ghost tier limits applied locally
Free user:     Purchases.getCustomerInfo() → no 'pro' entitlement → free limits
Pro Trial:     entitlements.active['pro'] exists + isInIntroductoryPricePeriod
Pro:           entitlements.active['pro'] exists
```

## Key Decisions (Rationale)

| Decision | Rationale |
|---|---|
| Full offline for ghost users (not limited plan creation) | Progression calc is 5 lines of math. Limiting plan creation requires new schema concepts (nullable routine_id) — more work than just porting it. |
| Analytics chart deferred for guests | 160 lines of bucketing logic in Python. Defer to save scope. Empty state shown instead. |
| Ghost limits mirror Free tier (slightly less) | Prevents migration conflicts where ghost data exceeds entitlement after registration. |
| Trial hoarding prevention via plan date check at workout-start | Simpler than restricting plan creation during trial. User keeps current active plan; future plans lock on expiry. |
| RevenueCat over direct Play Billing integration | Handles receipt validation, webhook sync, entitlement management. Free up to $2.5K MRR. Easier to add iOS later. |
| Sentry over Firebase Crashlytics | Captures JS errors (not just native crashes). Better for React Native. GDPR-safe with `beforeSend`. |
| Strong CSV as primary export format | De-facto standard. Hevy-importable. Human-readable in any spreadsheet. |
